    RAG Evaluator - RAG 评估器¶

    什么是 RAG 评估器？¶

    评估器是 RAG 系统的"质检员"，负责对检索结果和生成答案进行质量评估。就像一位严格的编辑，在文章发布前会检查内容是否准确、是否回答了问题、引用是否可靠。

    为什么需要评估器？¶

    RAG 系统面临三大核心问题：

    flowchart LR A[RAG 系统] --> B[检索质量] A --> C[生成质量] A --> D[幻觉问题] B --> B1["召回无关内容？"] C --> C1["答案不完整？"] D --> D1["胡编乱造？"] style D1 fill:#ffcdd2

    没有评估器，我们无法量化这些问题，也无法判断优化是否有效。

    一句话理解¶

    评估器 = 用 LLM 作为裁判，对检索和生成进行多维度打分

    为什么需要评估？¶

    RAG 系统的质量挑战¶

    挑战 说明 影响
    幻觉 LLM 生成的内容与上下文不符 用户信任丧失
    答非所问 答案没有回应用户的问题 用户体验差
    检索噪音 召回无关或低质量的文档 上下文污染
    效果不可知 不知道优化是否真的有效 盲目迭代

    评估的价值¶

    没有评估 → 盲目优化 → 不知道是否变好
有评估 → 量化问题 → 针对性优化 → 持续改进

    核心评估指标¶

    GoRAG 的评估器基于 RAGAS（Retrieval Augmented Generation Assessment）标准，提供三大核心指标：

    1. Faithfulness（忠诚度/幻觉检验）¶

    定义：生成的答案是否完全基于检索到的上下文，没有幻觉或外部知识。

    为什么重要？

    Faithfulness 是 RAG 系统的生命线。如果答案包含上下文没有的信息，用户会收到错误信息，可能做出错误的决策。

    评分标准：

    分数 含义 示例
    1.0 完全忠实 所有陈述都可从上下文推断
    0.5 部分忠实 一半陈述有依据，一半是幻觉
    0.0 完全幻觉 答案与上下文无关

    评估示例：

    上下文：爱因斯坦1905年发表了狭义相对论。
答案：爱因斯坦1905年获得了诺贝尔物理学奖。

评估：0.0（答案中的"诺贝尔物理学奖"在上下文中不存在）

    2. Answer Relevance（答案相关性）¶

    定义：生成的答案是否直接回答了用户的问题。

    为什么重要？

    即使答案完全基于上下文，如果答非所问，用户仍然无法得到有价值的信息。

    评分标准：

    分数 含义 示例
    1.0 完美回答 直接、完整地回答问题
    0.5 部分相关 回答了一些内容，但偏离重点
    0.0 答非所问 完全回避问题或回答无关内容

    评估示例：

    问题：如何申请退货？
答案：我们的产品都是经过严格质检的，品质有保障。

评估：0.0（答案完全没有回答退货问题）

    3. Context Precision（上下文精确度）¶

    定义：检索到的上下文中，有用信息的比例和位置。

    为什么重要？

    检索结果排在最前面的应该是最相关的。如果最相关的文档被埋在最后，会影响生成质量。

    评分标准：

    分数 含义 示例
    1.0 完美排序 最相关信息在顶部，无噪音
    0.5 中等质量 相关信息分散，混有噪音
    0.0 完全无关 检索到的都是无用信息

    评估类型详解¶

    1. 标准 RAG 评估 (RAGEvaluator)¶

    对完整的 RAG 流程进行评估，包括检索和生成两个阶段。

    flowchart LR A[Query + Chunks + Answer] --> B[RAGEvaluator] B --> C[Faithfulness] B --> D[Answer Relevance] B --> E[Context Precision]

    使用场景： - 离线批量评估 - A/B 测试对比 - 持续监控

    2. CRAG 评估 (CRAGEvaluator)¶

    专门为 CRAG 系统设计，判断检索结果是否需要回退到外部搜索。

    评估标签：

    标签 值 含义 行动策略
    Relevant 1 检索结果高度相关 直接用于生成
    Ambiguous 0 部分相关 优化查询 + DocStore 增强
    Irrelevant -1 不相关 切换到 Web 搜索

    工作流程：

    flowchart TD A[检索结果] --> B[CRAGEvaluator] B --> C{评估结果} C -->|Relevant| D[直接生成] C -->|Ambiguous| E[重新检索] C -->|Irrelevant| F[触发 Web 搜索] D --> G[最终答案] E --> G F --> G

    3. RAGAS LLM Judge¶

    基于 RAGAS 论文的标准实现，使用 LLM 作为评估器。

    优势： - 符合学术标准 - 无需人工标注 - 可自动化

    在 GoRAG 中使用评估器¶

    基本用法¶

    import "github.com/DotNetAge/gorag/pkg/generation/evaluator"

// 创建 RAGAS Judge
judge := evaluator.NewRagasLLMJudge(gpt4Client)

// 评估忠诚度
faithfulnessScore, reason, err := judge.EvaluateFaithfulness(
    ctx,
    query,        // 用户问题
    chunks,       // 检索到的文档块
    answer,       // 生成的答案
)

// 评估答案相关性
relevanceScore, reason, err := judge.EvaluateAnswerRelevance(
    ctx,
    query,
    answer,
)

// 评估上下文精确度
precisionScore, reason, err := judge.EvaluateContextPrecision(
    ctx,
    query,
    chunks,
)

fmt.Printf("Faithfulness: %.2f\n", faithfulnessScore)
fmt.Printf("Relevance: %.2f\n", relevanceScore)
fmt.Printf("Precision: %.2f\n", precisionScore)

    CRAG 评估¶

    import "github.com/DotNetAge/gorag/pkg/retrieval/crag"

evaluator := crag.NewEvaluator(myLLM)

// 评估检索结果
label, score, err := evaluator.Evaluate(ctx, query, chunks)

switch label {
case crag.Relevant:
    // 直接生成
case crag.Ambiguous:
    // 重新检索
case crag.Irrelevant:
    // 触发 Web 搜索
}

    批量评估¶

    // 加载测试数据集
dataset := loadTestDataset("./test_data.json")

var totalFaithfulness, totalRelevance, totalPrecision float64
for _, sample := range dataset {
    f, _, _ := judge.EvaluateFaithfulness(ctx, sample.Query, sample.Chunks, sample.Answer)
    r, _, _ := judge.EvaluateAnswerRelevance(ctx, sample.Query, sample.Answer)
    p, _, _ := judge.EvaluateContextPrecision(ctx, sample.Query, sample.Chunks)

    totalFaithfulness += f
    totalRelevance += r
    totalPrecision += p
}

n := float64(len(dataset))
fmt.Printf("Average Faithfulness: %.3f\n", totalFaithfulness/n)
fmt.Printf("Average Relevance: %.3f\n", totalRelevance/n)
fmt.Printf("Average Precision: %.3f\n", totalPrecision/n)

    评估指标计算¶

    RAGAS 综合分数¶

    RAGAS 总分是多个指标的加权平均：

    RAGAS_Score = 0.4 × Faithfulness
            + 0.3 × AnswerRelevance
            + 0.2 × ContextPrecision
            + 0.1 × ContextRecall

    权重调整建议¶

    场景 Faithfulness AnswerRel ContextPrec ContextRecall
    医疗问答 0.5 0.2 0.2 0.1
    客服对话 0.2 0.5 0.2 0.1
    研究报告 0.3 0.2 0.3 0.2
    创意写作 0.1 0.3 0.3 0.3

    适用场景¶

    ✅ 适合使用评估器的场景¶

    场景 说明 评估重点
    离线批量评估 发布前全面验证 全部指标
    A/B 测试 对比不同配置效果 差异指标
    持续监控 线上实时监控 Faithfulness
    问题诊断 定位问题根因 全部指标

    ❌ 不适合使用评估器的场景¶

    场景 原因 推荐替代
    实时性要求高 LLM 评估有延迟 采样评估
    成本敏感 LLM 调用成本高 减少评估频率

    实现详解¶

    接口定义¶

    // 评估器接口
type Evaluator interface {
    // Evaluate 执行完整评估
    Evaluate(ctx context.Context, query string, chunks []*Chunk, answer string) (*EvaluationResult, error)

    // EvaluateFaithfulness 评估忠诚度
    EvaluateFaithfulness(ctx context.Context, query string, chunks []*Chunk, answer string) (float32, string, error)

    // EvaluateAnswerRelevance 评估答案相关性
    EvaluateAnswerRelevance(ctx context.Context, query string, answer string) (float32, string, error)

    // EvaluateContextPrecision 评估上下文精确度
    EvaluateContextPrecision(ctx context.Context, query string, chunks []*Chunk) (float32, string, error)
}

// 评估结果
type EvaluationResult struct {
    Faithfulness      float32
    AnswerRelevance   float32
    ContextPrecision  float32
    Reasons           map[string]string
}
