    智能代理检索器¶

    更新摘要 - 新增独立的agentic包中的LLM分类器实现，提供更精确的意图分类 - 增强智能代理检索器功能，支持会话记忆和代理步骤跟踪 - 改进路由决策机制，增强元数据管理和错误处理 - 扩展意图类型支持，新增Relational和Global两类意图 - 新增合规性测试套件，确保路由执行失败时的自动回退机制 - 增强代理检索器的会话ID传递和步骤跟踪功能

    简介¶

    本文件面向GoRAG中的"智能代理检索器"（Agentic RAG），系统化阐述其设计理念、实现机制与工程实践。重点包括： - 新增agentic分类器：独立的LLM驱动意图分类器，提供更精确的查询意图识别 - 意图分类器与路由算法：如何基于查询意图自动选择最优检索策略 - 多策略协同工作：向量检索、图谱检索、全局检索等策略的组合与权衡 - 路由配置与策略权重：如何设置映射表、置信度阈值与默认策略 - 决策流程可视化：路由决策树与策略选择示例 - 性能监控与效果评估：可观测性指标与评估方法 - 新增合规性测试：确保路由执行失败时的自动回退机制

    项目结构¶

    围绕Agentic RAG的关键模块分布如下： - agentic包：独立的意图分类器、代理检索器和智能路由实现 - 检索器层：原生检索器、图谱检索器、智能路由检索器、代理检索器 - 查询增强与工具：意图分类、查询改写、子查询分解、过滤抽取、上下文裁剪 - 核心类型与接口：查询、意图类型、检索上下文、结果模型等 - 观测与日志：分布式追踪、指标采集、结构化日志

    核心组件¶

    • agentic分类器：独立的LLM驱动意图分类器，支持Chat、FactCheck、Relational、DomainSpecific四种意图类型
    • 智能路由检索器：根据意图分类结果将查询分发到不同检索器，并在失败时回退到默认策略
    • 意图分类器：对用户查询进行意图识别，输出意图类型、置信度与理由
    • 代理检索器：将检索过程委托给Agent，执行推理与工具调用，收集步骤信息并返回答案
    • 原生检索器：标准向量检索+生成的流水线实现
    • 图谱检索器：基于Cypher模板的实体关系深度检索
    • 查询增强工具：改写、分解、过滤抽取、上下文裁剪等
    • 新增合规性测试：确保路由执行失败时的自动回退机制

    架构总览¶

    Agentic RAG通过"意图驱动的智能路由"将查询分派至最适合的检索策略，同时结合代理推理能力实现自适应检索与答案生成。

    详细组件分析¶

    agentic分类器（LLM驱动意图分类）¶

    • 设计目标：提供独立的、精确的意图分类能力，支持四种专业意图类型
    • 功能特性
    • 支持Chat、FactCheck、Relational、DomainSpecific四种意图类型
    • 固定置信度0.9，提供稳定的分类结果
    • 当LLM客户端不可用时，回退到Chat意图
    • 简化的分类逻辑，直接解析LLM响应中的意图关键词
    • 实现要点
    • 使用预定义的提示词模板进行意图分类
    • 通过字符串匹配识别意图类型
    • 返回固定的置信度和分类理由
    flowchart TD Start(["开始"]) --> Check{"LLM客户端可用？"} Check -- 否 --> Fallback["回退到Chat意图<br/>置信度=1.0"] Check -- 是 --> Prompt["生成分类提示词"] Prompt --> Chat["调用LLM进行分类"] Chat --> Parse["解析响应内容"] Parse --> Intent{"识别意图类型"} Intent --> Return["返回分类结果"] Fallback --> Return

    智能路由检索器（Intent-Based Routing）¶

    • 设计目标：根据查询意图自动选择最优检索策略，提升准确率与效率
    • 关键流程 1) 意图分类：优先使用agentic分类器，失败时回退到查询分类器 2) 映射分发：依据映射表将查询分派到对应检索器；若无映射则回退默认策略 3) 结果聚合：附加路由元数据（意图、置信度、理由）后返回
    • 错误处理：分类失败或目标检索器执行失败时，记录告警并回退到默认检索器
    • 增强功能：新增执行级回退机制，确保系统稳定性

    意图分类器（Intent Classifier）¶

    • 功能：将用户查询归类到以下意图之一：聊天、领域知识、关系型、全局、事实核查
    • 输出：意图类型、置信度、理由
    • 配置项：提示词模板、默认意图、最低置信度阈值、日志与可观测性采集器
    • 容错：当LLM响应无法解析或置信度低于阈值时，回退到默认意图

    代理检索器（Agentic RAG）¶

    • 设计理念：将Agent作为"智能执行器"，在检索过程中进行推理、工具调用与步骤记录
    • 实现要点
    • 将Agent封装为流水线步骤，执行Chat并收集Agent步骤
    • 在检索上下文中维护Agentic状态，便于后续增强与溯源
    • 新增功能：支持会话ID传递，实现跨查询的上下文记忆
    • 增强功能：改进代理步骤跟踪，将步骤信息附加到元数据中
    • 返回答案与步骤元数据，便于调试与审计

    原生检索器（Native RAG）¶

    • 采用向量检索+生成的流水线，适合通用语义搜索
    • 支持可插拔的向量库、嵌入模型与LLM
    • 提供默认实现与选项配置

    图谱检索器（Graph RAG）¶

    • 基于Cypher模板的实体关系检索，适合复杂关系推理
    • 从上下文提取实体，执行模板查询并将结果注入上下文供LLM使用

    查询增强工具链¶

    • 查询改写：提升向量检索质量
    • 查询分解：复杂问题拆分为子查询
    • 过滤抽取：从自然语言中抽取显式过滤条件
    • 上下文裁剪：按相关性评分与Token预算裁剪检索上下文

    合规性测试套件¶

    • 执行失败回退测试：验证路由执行失败时自动回退到默认策略
    • 无匹配意图测试：验证无匹配意图时使用默认策略
    • 元数据保留测试：确保原始意图信息在回退过程中得到保留

    依赖分析¶

    • 接口与类型：统一的Retriever、IntentClassifier、Query、RetrievalResult等定义确保各组件松耦合
    • 检索上下文：贯穿整个检索流程的状态容器，承载中间结果与扩展字段
    • 组件耦合：路由检索器依赖意图分类器与多个具体检索器；代理检索器依赖Agent与流水线
    • agentic包独立性：agentic分类器和路由器形成独立的分类体系，可与查询分类器并存
    • 测试完整性：新增合规性测试确保路由机制的可靠性

    性能考虑¶

    • 并行与流水线：各检索器均采用流水线与可选并发策略，减少端到端延迟
    • Token控制：上下文裁剪按相关性与Token预算裁剪，降低LLM输入开销
    • 观测性：内置分布式追踪与指标采集，定位瓶颈与异常
    • 回退策略：路由失败时快速回退默认检索器，保障可用性
    • agentic分类器优势：固定置信度和简化的分类逻辑，提供稳定的性能表现
    • 增强的错误处理：执行级回退机制减少系统级故障影响

    故障排查指南¶

    • agentic分类器失败
    • 现象：使用查询分类器作为回退方案
    • 排查：检查LLM可用性、提示词模板、客户端配置
    • 意图分类失败
    • 现象：路由记录告警并回退默认策略
    • 排查：检查LLM可用性、提示词模板、最小置信度阈值
    • 目标检索器执行失败
    • 现象：返回错误并标注路由意图与置信度
    • 排查：确认目标检索器配置、向量库/图库连接、参数设置
    • 代理检索器无答案或步骤缺失
    • 现象：结果缺少agent_steps或答案为空
    • 排查：验证Agent可用性、历史消息传递、流水线步骤顺序
    • 会话ID传递问题
    • 现象：跨查询上下文记忆失效
    • 排查：确认查询元数据中session_id字段正确传递
    • 路由回退机制失效
    • 现象：路由执行失败时未回退到默认策略
    • 排查：运行合规性测试，检查路由逻辑中的错误处理

    结论¶

    Agentic RAG通过"意图驱动的智能路由"与"代理推理执行器"的组合，实现了对复杂查询的自适应检索与高阶推理。新增的独立agentic分类器提供了更精确的意图识别能力，而增强的代理检索器支持会话记忆和详细的步骤跟踪。最新增强包括：执行级回退机制确保系统稳定性、合规性测试验证路由可靠性、改进的元数据管理。配合完善的查询增强工具链与可观测性体系，能够在生产环境中稳定、高效地提供高质量检索与问答服务。

    附录¶

    路由配置与策略权重设置指南¶

    • 映射表构建
    • 建议：为常见意图建立明确映射，如"领域知识"→原生检索器，"关系型"→图谱检索器，"全局/事实核查"→代理检索器
    • 注意：避免重复映射同一意图，保持策略边界清晰
    • 新增建议：利用agentic分类器的Relational和FactCheck意图，分别映射到图谱检索器和代理检索器
    • 置信度阈值
    • 建议：默认不低于0.7；对高风险场景可提高阈值，降低误路由概率
    • agentic分类器特点：固定置信度0.9，稳定性高
    • 默认策略
    • 建议：优先选择"原生检索器"作为默认策略，兼顾速度与覆盖
    • 权重与融合
    • 对于多策略并行：结合相关性评分与Token预算进行融合（如RRF），并在路由阶段先做策略筛选
    • 错误处理策略
    • 建议：配置合理的超时时间和重试机制，确保系统在部分组件故障时仍能正常运行

    路由决策树与策略选择示例¶

    • 决策树
    • 输入：查询文本
    • 步骤：agentic分类器 → 查询分类器 → 查找映射 → 执行目标策略 → 附加元数据 → 返回
    • 异常：agentic分类器失败 → 查询分类器失败 → 回退默认策略
    • 示例
    • "公司政策"→领域知识→原生检索器
    • "某人与另一人的关系"→关系型→图谱检索器
    • "总结项目历史"→全局→代理检索器
    • "天气如何？"→事实核查→代理检索器（或外部搜索）
    • 新增示例："请分析公司的组织架构"→Relational→图谱检索器
    • 新增示例："请验证这个信息的准确性"→FactCheck→代理检索器

    多策略协同的优势与挑战¶

    • 优势
    • 提升准确性：针对不同意图选择最适合的策略
    • 增强鲁棒性：任一策略失败可回退
    • 可扩展性：新增策略无需改动现有逻辑
    • agentic分类器优势：提供稳定的分类结果，减少路由不确定性
    • 增强的错误处理：执行级回退机制确保系统稳定性
    • 挑战
    • 路由准确性：意图分类错误会导致策略误配
    • 成本控制：多策略并行可能增加延迟与Token消耗
    • 配置复杂度：映射表与阈值需要持续优化
    • agentic分类器集成：需要协调两种分类器的使用策略
    • 测试覆盖：需要充分的测试确保各种异常情况下的正确行为

    性能监控与效果评估方法¶

    • 监控指标
    • 意图分类耗时、路由命中率、各策略命中数与成功率
    • 检索耗时、Token用量、答案生成耗时
    • agentic分类器指标：分类成功率、响应时间、回退频率
    • 路由机制指标：回退次数、回退成功率、平均恢复时间
    • 评估方法
    • RAG评估：基于事实一致性、相关性与总体分数
    • 自我评估：结合Self-RAG的反思与纠错能力
    • A/B测试：对比启用路由前后的效果差异
    • agentic分类器评估：比较agentic分类器与查询分类器的准确性差异
    • 合规性测试：定期运行路由回退测试确保系统可靠性
    • 可观测性增强
    • 增加路由决策日志，记录意图类型、置信度和决策原因
    • 监控会话ID传递情况，确保上下文连续性
    • 跟踪代理步骤执行情况，便于调试和优化
