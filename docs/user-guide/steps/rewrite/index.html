    Rewrite Step - 查询重写步骤¶

    是什么¶

    Rewrite Step 是 Pipeline 中的查询处理组件，负责对用户查询进行重写和优化，提升检索效果。

    核心功能¶

    • 消除查询歧义：将模糊表述改为清晰描述
    • 扩展关键词：补充同义词和相关术语
    • 去除冗余表达：移除无意义的闲聊词汇
    • 指代消解：将代词解析为具体实体

    核心原理¶

    flowchart LR A["口语查询: Go 咋学啊？"] --> B[Rewrite Step] B --> C["标准查询: 如何学习 Go 语言？"]

    工作流程¶

    flowchart TD A[原始查询] --> B[LLM 分析意图] B --> C[应用重写规则] C --> D[优化表达] D --> E[返回重写后的查询]

    使用示例¶

    基本用法¶

    import (
    "github.com/DotNetAge/gorag/pkg/core"
    "github.com/DotNetAge/gorag/pkg/steps/rewrite"
)

rewriteStep := rewrite.NewQueryRewriter(llm)

state := &core.RetrievalContext{
    Query: &core.Query{
        Text: "Go 咋样？",
    },
}

err := rewriteStep.Execute(ctx, state)
println("重写后:", state.Query.Text)

    Pipeline 中使用¶

    p := pipeline.New[*core.RetrievalContext]()

p.AddStep(rewrite.NewQueryRewriter(llm))
p.AddStep(vector.NewSearchStep(vecStore))

p.Execute(ctx, state)

    效果示例¶

    原始查询 重写后
    "Go 咋样？" "Go 语言的优势和特点"
    "ML 的原理" "机器学习的原理"
    "GPT 和 BRT 的区别" "GPT 和 BERT 的区别"
    "问一下关于 AI 的事情" "人工智能的定义和应用"
    "那个公司..." "Apple 公司..."

    配置选项¶

    选项 类型 默认值 说明
    WithLLM chat.Client 必需 用于重写的 LLM

    适用场景¶

    ✅ 适合使用¶

    • 用户查询表述不清
    • 包含口语化表达
    • 包含缩写或简称
    • 需要关键词扩展
    • 客服场景

    ❌ 不适合使用¶

    • 简单明确的查询
    • 已有高质量查询
    • 延迟敏感场景

    注意事项¶

    1. LLM 质量：重写效果依赖 LLM 的语言理解能力
    2. 延迟增加：增加了 LLM 调用延迟
    3. 过度重写：有时重写会改变原意，注意审核
