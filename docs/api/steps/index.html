    管道步骤API¶

    本文件面向 GoRAG 的“管道步骤”API，系统化梳理检索增强生成（RAG）流程中可插拔的步骤模块，覆盖以下方面： - 步骤接口定义与职责边界 - 每个步骤的功能、输入输出、配置参数与行为约束 - 执行顺序与依赖关系 - 可组合性与扩展机制 - 错误处理策略与性能考量 - 自定义步骤的开发指南与最佳实践

    项目结构¶

    GoRAG 将“检索”和“生成”两大阶段通过统一的管道上下文进行串联，并在检索阶段提供一系列可插拔的“步骤”，用于查询预处理、结果融合、重排序、压缩等。

    核心组件¶

    • 管道上下文与状态
    • 检索上下文包含查询、检索结果分组、过滤器、原始查询备份、以及“智能体”元信息（如子查询、假设文档、是否命中缓存等）。
    • 生成上下文在检索完成后，将扁平化的块集合与查询传递给生成器，产出最终答案。
    • 核心接口
    • 检索器、生成器、重排序器、查询分解器、过滤器抽取器、意图分类器、融合引擎等，均以接口形式解耦具体实现，便于替换与扩展。

    详细组件分析¶

    语义缓存步骤（SemanticCache）¶

    • 功能
    • 在检索前检查是否存在语义等价的历史答案；若命中则直接填充答案并标记缓存命中；否则放行后续检索。
    • 输入
    • 查询对象（必须非空）
    • 输出
    • 若命中：在上下文中写入答案并标记缓存命中；否则不改变上下文
    • 配置参数
    • 缓存实现（实现语义缓存接口）、日志器、指标收集器
    • 行为与错误处理
    • 查询为空时跳过；缓存检查失败时静默忽略并继续流程
    • 性能影响

    • 显著减少重复查询的检索与生成开销

    • cache.go:57-78

    查询改写（QueryRewrite）¶

    • 功能
    • 利用 LLM 将模糊或歧义查询改写为更清晰、更具检索表达力的版本
    • 输入
    • 查询文本（非空）
    • 输出
    • 更新后的查询文本；同时保留原始查询以便溯源
    • 配置参数
    • LLM 客户端、日志器、指标收集器
    • 行为与错误处理
    • 改写失败或返回空时，保留原查询并记录警告
    • 性能影响
    • 单次 LLM 调用成本较高，建议配合缓存使用

    查询分解（QueryDecomposition）¶

    • 功能
    • 将复杂查询拆分为若干子查询，便于多路检索与聚合
    • 输入
    • 查询对象（必须非空）
    • 输出
    • 子查询列表与复杂度标记；写入智能体上下文
    • 配置参数
    • 分解器实现、日志器
    • 行为与错误处理
    • 查询为空时报错；分解失败时报错
    • 性能影响
    • 增加检索次数，但提升召回质量

    过滤条件提取（FilterFromQuery）¶

    • 功能
    • 从自然语言查询中抽取结构化过滤键值对，供检索阶段使用
    • 输入
    • 查询对象（必须存在）
    • 输出
    • 写入上下文的 Filters；同时写入智能体元信息
    • 配置参数
    • 过滤器抽取器实现
    • 行为与错误处理
    • 查询缺失时报错；抽取失败时报错
    • 性能影响
    • 降低向量检索范围，提高效率

    假设文档生成（HyDE）¶

    • 功能
    • 基于查询生成“假设文档”，用于引导向量检索到更相关的结果
    • 输入
    • 查询对象（必须非空）
    • 输出
    • 在智能体上下文中记录假设文档与标记
    • 配置参数
    • 生成器实现、日志器
    • 行为与错误处理
    • 查询为空时报错；生成失败时报错
    • 性能影响
    • 额外一次生成调用，换取更好的检索质量

    步退生成（StepBack）¶

    • 功能
    • 将具体查询抽象为更高层、更通用的问题，再回填为新的查询
    • 输入
    • 查询对象（必须存在）
    • 输出
    • 新的高层查询文本；写入智能体元信息
    • 配置参数
    • 分解器实现（复用查询分解接口）
    • 行为与错误处理
    • 查询缺失时报错；分解失败时报错
    • 性能影响
    • 两次分解调用（一次高层，一次具体），适合复杂领域

    重排序（Rerank）¶

    • 功能
    • 使用交叉编码模型对检索块重新打分与排序，选取前 K
    • 输入
    • 查询对象、检索块（二维分组）
    • 输出
    • 更新后的检索块（仅保留重排后前 K）
    • 配置参数
    • 重排序器实现、topK、日志器、指标收集器
    • 行为与错误处理
    • 无块或查询缺失时跳过；失败时报错
    • 性能影响
    • 重排序计算成本高，建议结合过滤与压缩使用

    上下文压缩（ContextCompression）¶

    • 功能
    • 将大量检索块压缩为一条高度相关的信息，作为最终答案
    • 输入
    • 查询文本、检索块
    • 输出
    • 替换答案为压缩结果
    • 配置参数
    • LLM 客户端、日志器、指标收集器、最大输出 token 数
    • 行为与错误处理
    • 无块或查询为空时跳过；LLM 调用失败记录错误并报错
    • 性能影响
    • 一次 LLM 调用，显著减少生成器输入规模

    文档增强（DocStoreEnrichment）¶

    • 功能
    • 对已检索到的块附加其父文档内容，增强生成器上下文
    • 输入
    • 检索块（需包含文档 ID）
    • 输出
    • 将父文档内容写入块元数据
    • 配置参数
    • 文档存储实现、日志器
    • 行为与错误处理
    • DocStore 为空时跳过；获取失败记录警告并继续
    • 性能影响
    • 额外的存储读取，建议按需启用

    答案生成（Generator）¶

    • 功能
    • 基于查询与检索块生成最终答案
    • 输入
    • 查询对象、检索块（扁平化）
    • 输出
    • 答案对象写入上下文
    • 配置参数
    • 生成器实现、日志器、指标收集器
    • 行为与错误处理
    • 查询缺失时报错；生成失败时报错
    • 性能影响
    • 一次 LLM 调用，决定最终质量与成本

    索引阶段步骤（Indexing Steps）¶

    • 功能概览
    • 文件发现、多解析器流式解析、语义分块、批量嵌入、向量入库、实体抽取等
    • 典型组合
    • 发现 → 多解析 → 语义分块 → 批量嵌入 → 向量入库
    • 行为与错误处理
    • 各步骤独立，遵循 gochat 管道约定；失败时中断并返回错误
    • 性能影响
    • I/O 与嵌入成本为主，建议批量化与并发化

    依赖分析¶

    • 步骤与接口
    • 所有步骤均依赖统一的管道 Step 接口与核心上下文类型
    • 步骤通过注入核心接口（如 Generator、Reranker、FilterExtractor 等）实现功能
    • 步骤间依赖
    • 查询改写与分解通常在检索前执行，为后续检索提供更清晰的查询
    • 过滤条件提取与假设文档生成可并行或先后执行，视场景而定
    • 重排序与上下文压缩通常在检索后、生成前执行
    • 外部依赖
    • LLM 客户端、向量存储、文档存储、指标收集器等由上层装配注入

    性能考虑¶

    • 并发与批处理
    • 索引阶段建议批量嵌入与并发解析；检索阶段可并行多路检索
    • 成本控制
    • 优先使用语义缓存；在生成前使用上下文压缩；谨慎使用昂贵的重排序
    • I/O 优化
    • 合理设置 topK 与过滤条件，减少向量检索与存储读取
    • 指标监控
    • 通过指标接口记录检索耗时、结果数量、错误率与嵌入计数，辅助容量规划与调优

    故障排查指南¶

    • 常见错误来源
    • 查询缺失或为空：多数步骤在查询缺失时会报错或跳过，需检查上游步骤
    • LLM 调用失败：改写、压缩、生成等步骤在 LLM 失败时会报错，需检查凭据与网络
    • 重排序或压缩失败：记录错误并中断，需检查模型服务与输入格式
    • 建议排查路径
    • 开启详细日志，定位失败步骤
    • 回放最小化输入，验证单步骤行为

    • 检查外部服务可用性与配额限制

    结论¶

    GoRAG 的管道步骤以接口解耦、以上下文驱动，提供了从查询预处理、检索、重排序、压缩到最终生成的完整链路。通过合理编排与参数调优，可在保证质量的同时控制成本与延迟。建议在生产环境中结合缓存、过滤与压缩策略，并持续通过指标监控优化整体性能。

    附录：自定义步骤开发指南与最佳实践¶

    • 设计原则
    • 单一职责：每个步骤只做一件事，保持可测试与可替换
    • 明确输入输出：严格校验上下文字段，必要时提供默认值或跳过逻辑
    • 可观测性：记录关键事件与指标，便于追踪与诊断
    • 实现要点
    • 实现 Step 接口，提供名称与执行函数
    • 注入所需核心接口（如 Generator、Reranker、Client 等）
    • 对外部调用进行错误包装与降级处理
    • 组合与编排
    • 使用管道 AddSteps 顺序组织步骤；复杂流程可拆分为子管道
    • 注意步骤顺序：改写与分解通常在前，过滤与假设文档在检索前，重排序与压缩在检索后
    • 参数与配置
    • 为步骤提供合理的默认值与最小可用配置，避免运行时 panic
    • 通过选项函数或构造器集中初始化，便于测试与替换
    • 测试建议
    • 为每个步骤编写单元测试，模拟成功与失败分支
    • 使用假实现替换外部依赖，确保测试稳定
