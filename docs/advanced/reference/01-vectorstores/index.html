    向量库对比¶

    什么是向量库？¶

    向量库（Vector Store）是专门用于存储和检索高维向量的数据库。在 RAG 系统中，向量库存储文档的向量表示，支持通过向量相似度快速找到与查询最相关的文档。

    一句话理解¶

    向量库 = 专门存储"文本向量"的数据库，支持快速相似度搜索

    为什么需要向量库？¶

    传统数据库按精确匹配查询（如 WHERE name = "张三"），而向量库按相似度查询（如 找出与"张三"最相似的 10 个人）。

    查询类型 示例 传统数据库 向量库
    精确匹配 找"张三" ✅ 高效 ❌ 不擅长
    相似搜索 找"类似张三的人" ❌ 无法实现 ✅ 高效

    为什么需要向量库？¶

    海量文本的相似度搜索¶

    RAG 系统需要从海量文档中找到与用户问题最相关的片段：

    用户问题："如何学习 Go 语言？"
        ↓
    计算问题向量
        ↓
    向量库检索：找出最相似的文档
        ↓
    返回 Top-K 最相关文档

    如果没有向量库，就需要逐个文档比较，计算复杂度为 O(n)；有了向量库，可以通过 ANN（近似最近邻）算法将复杂度降到 O(log n) 或更低。

    向量库的核心能力¶

    能力 说明
    向量存储 存储高维向量
    相似度检索 快速找到最相似的向量
    批量查询 支持批量检索
    元数据过滤 支持附加条件过滤
    索引支持 HNSW、IVF 等高效索引

    对比总览¶

    向量库 部署方式 外部依赖 扩展性 推荐场景
    GoVector 本地 无 单机 快速原型、中小规模生产
    Milvus 混合 需要 分布式 超大规模、云原生

    Weaviate 混合 需要 分布式 混合搜索（向量+关键词）

    GoVector（推荐本地）¶

    为什么选择 GoVector？¶

    GoVector 是 GoRAG 内置的纯 Go 向量库，遵循 GoRAG "零外部依赖"的设计理念：

    特点 说明
    纯 Go 实现 无需安装额外服务
    零外部依赖 数据完全本地存储
    性能优异 自带百万级 benchmark
    无缝集成 与 GoRAG 其他组件完美配合

    性能数据¶

    GoVector Benchmark (百万级向量):
- 索引构建: 2.3 秒
- 单次查询: 0.8 毫秒
- 召回率: 95%+ (HNSW)

    使用示例¶

    import "github.com/DotNetAge/gorag/pkg/indexing/vectorstore/govector"

store, err := govector.NewStore(
    govector.WithDimension(1536),
    govector.WithIndexType(govector.HNSW),
)

err = store.Upsert(ctx, chunks)
results, err := store.Search(ctx, queryVector, 5)

    适用场景¶

    场景 推荐度 原因
    快速原型 ⭐⭐⭐⭐⭐ 零配置，开箱即用
    中小规模生产 ⭐⭐⭐⭐ 性能足够，运维简单
    数据敏感环境 ⭐⭐⭐⭐⭐ 完全本地，无数据外传
    超大规模 ⭐⭐ 建议使用分布式方案

    Milvus¶

    特点¶

    特点 说明
    分布式架构 支持水平扩展到万亿级向量
    丰富的索引类型 HNSW、IVF、PQ 等
    云原生 支持 K8s 部署
    活跃社区 成熟的开源项目

    使用示例¶

    import "github.com/DotNetAge/gorag/pkg/indexing/vectorstore/milvus"

store, err := milvus.NewStore(
    milvus.WithURI("tcp://localhost:19530"),
    milvus.WithDimension(1536),
)

    适用场景¶

    场景 推荐度 原因
    超大规模向量 ⭐⭐⭐⭐⭐ 分布式架构支持万亿级
    云原生部署 ⭐⭐⭐⭐ K8s 原生支持
    需要多种索引 ⭐⭐⭐⭐ 支持主流索引类型

    特点¶

    pinecone.WithEnvironment("us-east-1"),

    场景 推荐度 原因

    快速上线 ⭐⭐⭐⭐ 几分钟完成配置
    全球部署 ⭐⭐⭐⭐ 多区域支持

    适用场景¶

    场景 推荐度 原因
    高性能需求 ⭐⭐⭐⭐⭐ 业界领先性能

    常见问题¶

    Q: 向量维度不匹配会怎样？¶

    A：向量维度必须在创建存储时指定，后续无法更改。如果维度不匹配，存储或检索时会报错。

    // 创建时指定维度
store, _ := govector.NewStore(govector.WithDimension(1536))

// Embedder 输出必须也是 1536 维
embedder, _ := openai.NewEmbedder(...)  // text-embedding-ada-002 输出 1536 维

    Q: GoVector 能支持多少向量？¶

    A：GoVector 在单机环境下可以高效处理百万级向量。如果需要更大规模，建议使用 Milvus 等分布式方案。

    Q: 如何迁移到其他向量库？¶

    A：GoRAG 提供了统一的向量库接口，可以按需切换：

    // 从 GoVector 切换到 Milvus
store, err := milvus.NewStore(
    milvus.WithURI("tcp://localhost:19530"),
    milvus.WithDimension(1536),
)
