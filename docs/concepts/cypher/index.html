    Cypher 查询语言与图检索¶

    什么是 Cypher？¶

    Cypher 是 Neo4j 图数据库的查询语言，用于处理"节点"和"关系"而不是传统的"表"和"行"。如果说 SQL 是用来操作关系型数据库的，那么 Cypher 就是用来操作图数据库的。

    一句话理解¶

    Cypher = 操作"图"的查询语言，让 RAG 具备关系推理能力

    Cypher 在 RAG 中的位置¶

    flowchart LR A[用户查询] --> B[向量检索] B --> C[返回相关文档] D[用户查询] --> E[Cypher 查询]:::green E --> F[图数据库]:::lightgreen F --> G[关系路径] G --> H[结构化知识] H --> I[增强上下文] classDef green fill:#e8f5e9 classDef lightgreen fill:#c8e6c9

    为什么需要 Cypher？¶

    向量检索的局限¶

    局限 说明 示例
    语义匹配 只能找到"相似"的内容 "苹果"可能召回水果或公司
    关系推理 无法理解实体间的关系 无法回答"CEO 的公司"类型问题
    多跳查询 一次查询难以跨越多层关系 "张三公司CEO的邻居"

    Cypher 的优势¶

    优势 说明 示例
    关系推理 支持多跳路径查询 MATCH (a)-[:WORKS_AT]->(b)<-[:CEO_OF]-(c)
    结构化过滤 支持属性条件筛选 WHERE p.age > 30 AND p.city = '北京'
    路径发现 找出实体间的关联路径 找出"张三"和"李四"的关系链

    Cypher 基础语法¶

    1. 节点语法¶

    创建节点：

    // 创建一个 Person 节点
CREATE (p:Person {name: "张三", age: 30})

// 创建一个 Company 节点
CREATE (c:Company {name: "A公司", industry: "科技"})

    匹配节点：

    // 匹配所有 Person 节点
MATCH (p:Person)
RETURN p

// 匹配特定条件的节点
MATCH (p:Person {name: "张三"})
RETURN p

    2. 关系语法¶

    创建关系：

    // 创建节点和关系
CREATE (p:Person {name: "张三"})-[:WORKS_AT]->(c:Company {name: "A公司"})

    匹配关系：

    // 匹配工作关系
MATCH (p:Person)-[:WORKS_AT]->(c:Company)
WHERE p.name = "张三"
RETURN c.name

// 匹配任意关系
MATCH (p:Person)-[r]->(c:Company)
RETURN type(r), c.name

    3. 关系类型说明¶

    关系类型 含义 示例
    [:WORKS_AT] 工作于 张三 工作于 A公司
    [:CEO_OF] 是...的 CEO 李四 是 A公司 的 CEO
    [:FRIEND_OF] 是...的朋友 王五 是 张三 的朋友
    [:LOCATED_IN] 位于 A公司 位于 北京

    4. 路径查询¶

    单跳查询：

    // 查找张三工作的公司
MATCH (p:Person {name: "张三"})-[:WORKS_AT]->(c:Company)
RETURN c.name

    多跳查询：

    // 查找张三公司 CEO 的名字
MATCH (p:Person {name: "张三"})-[:WORKS_AT]->(c:Company)<-[:CEO_OF]-(ceo:Person)
RETURN ceo.name

    路径发现：

    // 查找任意两个节点间的路径
MATCH path = (a:Person {name: "张三"})-[*1..3]-(b:Person {name: "李四"})
RETURN path

    5. 条件过滤¶

    // 多条件过滤
MATCH (p:Person)
WHERE p.age > 30 AND p.city = "北京"
RETURN p.name, p.age

// 模糊匹配
MATCH (p:Person)
WHERE p.name CONTAINS "张"
RETURN p

// 正则匹配
MATCH (p:Person)
WHERE p.name =~ "张.*"
RETURN p

    Cypher 在 GraphRAG 中的应用¶

    1. 多跳推理 (Multi-hop Reasoning)¶

    问题：用户问"张三入职公司的 CEO 是谁？"

    这需要两步跳： 1. 找到张三工作的公司 2. 找到该公司的 CEO

    Cypher 解决方案：

    MATCH (p:Person {name: "张三"})-[:WORKS_AT]->(c:Company)<-[:CEO_OF]-(ceo:Person)
RETURN ceo.name AS ceo_name, c.name AS company_name

    执行流程：

    flowchart LR A[张三] --> B[WORK_AT: A公司] B --> C[CEO_OF: 李四] A --> C

    2. 社区摘要检索 (Community Summary)¶

    微软 GraphRAG 的核心概念，支持全局性问题：

    // 抓取指定层级的社区总结
MATCH (c:Community {level: 2})
RETURN c.summary, c.entity_count
ORDER BY c.entity_count DESC

// 抓取包含特定实体周边的社区
MATCH (c:Community)-[:CONTAINS]->(e:Entity)
WHERE e.name CONTAINS "人工智能"
RETURN c.summary

    3. 结构化过滤检索¶

    结合图属性和向量检索的优势：

    // 结合时间过滤的查询
MATCH (d:Document)
WHERE d.publish_date > '2024-01-01'
  AND d.topic = 'AI'
RETURN d.content, d.title

// 结合地理位置的查询
MATCH (e:Event)-[:LOCATED_IN]->(c:City {name: "北京"})
WHERE e.date > '2024-01-01'
RETURN e.name, e.description

    Text-to-Cypher 实践¶

    让 LLM 将自然语言转换为 Cypher 查询：

    1. 模板定义¶

    // CypherQueryTemplate 定义查询模板
type CypherQueryTemplate struct {
    // 实体类型定义
    EntityTypes []EntityType

    // 关系类型定义
    RelationTypes []RelationType

    // 查询示例（Few-shot）
    Examples []CypherExample
}

// EntityType 实体类型定义
type EntityType struct {
    Name        string            // 如 "Person", "Company"
    Properties  []string         // 如 ["name", "age", "city"]
    Description string            // 实体类型描述
}

// RelationType 关系类型定义
type RelationType struct {
    Type        string            // 如 "WORKS_AT"
    From        string            // 源实体类型
    To          string            // 目标实体类型
    Description string            // 关系描述
}

    2. LLM Prompt 模板¶

    你是一个图数据库查询专家。请根据用户问题，生成 Cypher 查询语句。

可用实体类型：
- Person: 人 {name, age, city}
- Company: 公司 {name, industry, location}
- Product: 产品 {name, category, price}

可用关系类型：
- WORKS_AT: 人 -> 公司 (某人工作于某公司)
- CEO_OF: 人 -> 公司 (某人是某公司CEO)
- PRODUCE: 公司 -> 产品 (某公司生产某产品)

用户问题：{user_question}

请生成 Cypher 查询：

    3. LLM 生成示例¶

    用户输入："苹果公司的 CEO 是谁？"

    LLM 生成的 Cypher：

    MATCH (c:Company {name: "苹果公司"})<-[:CEO_OF]-(ceo:Person)
RETURN ceo.name

    用户输入："张三工作的公司生产什么产品？"

    LLM 生成的 Cypher：

    MATCH (p:Person {name: "张三"})-[:WORKS_AT]->(c:Company)-[:PRODUCE]->(prod:Product)
RETURN prod.name, prod.category

    在 GoRAG 中使用 Cypher¶

    1. GraphStore 接口¶

    // GraphStore 图存储接口
type GraphStore interface {
    // Query 执行 Cypher 查询
    Query(ctx context.Context, cypher string, params map[string]interface{}) ([]map[string]interface{}, error)

    // CreateNode 创建节点
    CreateNode(ctx context.Context, label string, properties map[string]interface{}) error

    // CreateRelation 创建关系
    CreateRelation(ctx context.Context, fromID, toID string, relType string) error
}

    2. Neo4j 实现示例¶

    // Neo4jStore Neo4j 图存储实现
type Neo4jStore struct {
    driver *neo4j.Driver
}

// Query 执行 Cypher 查询
func (s *Neo4jStore) Query(ctx context.Context, cypher string, params map[string]interface{}) ([]map[string]interface{}, error) {
    session := s.driver.NewSession(neo4j.SessionConfig{})
    defer session.Close()

    result, err := session.Run(cypher, params)
    if err != nil {
        return nil, err
    }

    var records []map[string]interface{}
    for result.Next() {
        record := result.Record()
        r := make(map[string]interface{})
        for k, v := range record.GetByIndex(0).(map[string]interface{}) {
            r[k] = v
        }
        records = append(records, r)
    }

    return records, nil
}

    3. RAG Pipeline 集成¶

    // CypherRetrievalStep Cypher 检索步骤
type CypherRetrievalStep struct {
    graphStore GraphStore
    template   *CypherQueryTemplate
}

// Process 执行 Cypher 检索
func (s *CypherRetrievalStep) Process(ctx context.Context, query string, results []core.Result) ([]core.Result, error) {
    // 1. LLM 生成 Cypher
    cypher, err := s.generateCypher(query)
    if err != nil {
        return nil, err
    }

    // 2. 执行查询
    graphResults, err := s.graphStore.Query(ctx, cypher, nil)
    if err != nil {
        return nil, err
    }

    // 3. 转换为标准结果格式
    return s.convertToResults(graphResults), nil
}

    Cypher 模板库¶

    常用查询模板¶

    // 模板 1：查找实体的所有关系
MATCH (e {name: $entityName})-[r]-(connected)
RETURN labels(connected)[0] AS type, connected.name AS name, type(r) AS relation

// 模板 2：查找路径
MATCH path = (a {name: $from})-[*1..5]-(b {name: $to})
RETURN path

// 模板 3：社区发现
MATCH (e:Entity)-[:CONNECTS]-(c:Community)
WHERE e.name = $entityName
RETURN c.summary, c.level

// 模板 4：批量实体查询
MATCH (e:Entity)
WHERE e.name IN $entityNames
RETURN e.name, e.description

    总结¶

    能力 向量检索 Cypher 检索
    语义匹配 ✅ 擅长 ❌ 不擅长
    关系推理 ❌ 不擅长 ✅ 擅长
    多跳查询 ❌ 不擅长 ✅ 擅长
    属性过滤 ⚠️ 有限 ✅ 强大

    最佳实践：将向量检索和 Cypher 检索结合使用，让 RAG 系统既具备语义理解能力，又具备逻辑推理能力。
