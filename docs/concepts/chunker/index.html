    分块策略 (Chunking)¶

    什么是分块？¶

    分块是将长文档切分为较小文本片段的过程，是 RAG 系统中连接文档解析与向量化的关键环节。

    一句话理解¶

    分块 = 把"长文章"切成"小段落"，每个小段落都能独立表达一个完整的意思

    分块在 RAG 中的位置¶

    flowchart LR A[原始文档] --> B[文档解析] B --> C[纯文本] C --> D[数据清洗] D --> E[分块处理] E --> F[文本块] F --> G[向量化] G --> H[向量数据库] style E fill:#fff3e0

    为什么需要分块？¶

    不分块的问题¶

    问题 说明 示例
    向量稀释 长文档包含太多主题，向量表达模糊 整本书作为一块
    精度下降 检索返回整页，用户需要自己找答案 FAQ 整页返回
    Token 浪费 无关内容占用 LLM 上下文 不需要的内容
    重复存储 相同内容在不同位置重复出现 通用条款重复

    分块的核心目标¶

    目标 1：每个块语义完整，独立可检索
目标 2：块大小适中，平衡精度与上下文
目标 3：块之间有重叠，保持上下文连续性

    分块策略核心参数¶

    1. ChunkSize（分块大小）¶

    ChunkSize 是最重要的参数，决定每个块包含的字符/Token 数量。

    ChunkSize 特点 适用场景
    过小 (< 100字) 语义聚焦但上下文有限 精确问答
    适中 (300-500字) 平衡方案 通用场景
    偏大 (800-1500字) 上下文丰富 摘要、推理

    2. ChunkOverlap（分块重叠）¶

    相邻块之间保留的重叠内容，用于保持上下文连续性。

    flowchart LR A[块1: 内容...] --> B[块2: ...重叠...] B --> C[块3: ...重叠...]
    Overlap 优点 缺点
    过大 上下文连贯 重复内容多
    过小 独特内容多 可能丢失跨界信息

    分块策略详解¶

    1. 固定大小分块 (Fixed Size Chunking)¶

    最简单的分块方式，按固定字符数切分。

    // 固定大小分块实现
func FixedSizeChunking(text string, chunkSize, overlap int) []string {
    var chunks []string

    if len(text) <= chunkSize {
        return []string{text}
    }

    step := chunkSize - overlap

    for i := 0; i < len(text); i += step {
        end := i + chunkSize
        if end > len(text) {
            end = len(text)
        }

        chunk := text[i:end]
        chunks = append(chunks, chunk)

        if end == len(text) {
            break
        }
    }

    return chunks
}

    优点：简单、快速、可预测 缺点：可能在句子中间截断

    2. 句子级分块 (Sentence Chunking)¶

    按句子边界切分，保证每个块是完整的句子。

    // 句子级分块
func SentenceChunking(text string, maxSentence int) []string {
    sentences := SplitSentences(text)
    var chunks []string
    var current []string

    for _, sent := range sentences {
        current = append(current, sent)

        if len(current) >= maxSentence {
            chunks = append(chunks, strings.Join(current, ""))
            current = nil
        }
    }

    if len(current) > 0 {
        chunks = append(chunks, strings.Join(current, ""))
    }

    return chunks
}

    优点：句子完整、语义连贯 缺点：块大小不均匀

    3. 段落级分块 (Paragraph Chunking)¶

    按段落边界切分，保持语义单元的完整性。

    // 段落级分块
func ParagraphChunking(text string, maxParagraphs int) []string {
    // 按双换行符分割段落
    paragraphs := strings.Split(text, "\n\n")

    var chunks []string
    var current []string

    for _, para := range paragraphs {
        if para = strings.TrimSpace(para); para == "" {
            continue
        }

        current = append(current, para)

        if len(current) >= maxParagraphs {
            chunks = append(chunks, strings.Join(current, "\n\n"))
            current = nil
        }
    }

    if len(current) > 0 {
        chunks = append(chunks, strings.Join(current, "\n\n"))
    }

    return chunks
}

    优点：语义完整、可读性强 缺点：段落大小差异大

    4. 递归分块 (Recursive Chunking)¶

    按优先级尝试不同级别的分割，直到块大小合适。

    // 递归分块 - 按优先级尝试不同分割符
var separators = []string{
    "\n\n\n", // 标题后双空行
    "\n\n",   // 段落分隔
    "\n",     // 换行
    ". ",     // 句子
    ", ",     // 短语
    "",       // 最后按字符
}

func RecursiveChunking(text string, chunkSize int) []string {
    return recursiveSplit(text, 0, chunkSize)
}

func recursiveSplit(text string, sepIndex, chunkSize int) []string {
    if sepIndex >= len(separators) {
        // 最后一层，按字符数分割
        return fixedChunking(text, chunkSize)
    }

    separator := separators[sepIndex]
    if separator == "" {
        return fixedChunking(text, chunkSize)
    }

    parts := strings.Split(text, separator)

    var chunks []string
    var current string

    for _, part := range parts {
        test := current + separator + part
        if len(test) > chunkSize {
            if current != "" {
                chunks = append(chunks, strings.TrimSpace(current))
            }
            current = part
        } else {
            current = test
        }
    }

    if current != "" {
        // 最后一个块可能太小，递归处理
        if len(chunks) > 0 && len(current) < chunkSize/2 {
            // 合并到前一个块
            chunks[len(chunks)-1] += separator + current
        } else {
            chunks = append(chunks, strings.TrimSpace(current))
        }
    }

    return chunks
}

    优点：智能选择分割点、块大小可控 缺点：实现复杂

    5. 语义分块 (Semantic Chunking)¶

    基于语义相似度，在主题转换处切分。

    // 基于句子嵌入的语义分块
func SemanticChunking(text string, sentences []string, embeddings [][]float32, threshold float32) []string {
    var chunks []string
    var currentChunk []string

    for i, sent := range sentences {
        currentChunk = append(currentChunk, sent)

        // 计算与下一句的相似度
        if i < len(sentences)-1 {
            similarity := cosineSimilarity(embeddings[i], embeddings[i+1])

            // 相似度下降说明主题转换
            if similarity < threshold {
                chunks = append(chunks, strings.Join(currentChunk, ""))
                currentChunk = nil
            }
        }
    }

    if len(currentChunk) > 0 {
        chunks = append(chunks, strings.Join(currentChunk, ""))
    }

    return chunks
}

    优点：块内语义高度一致 缺点：需要调用嵌入模型、成本高

    目标 保留完整上下文

    场景 C：代码检索¶

    配置 值
    分块方式 AST 语法树
    ChunkSize 按函数/类边界
    ChunkOverlap 保留函数签名
    目标 保持代码完整性

    分块质量评估¶

    1. 块内聚合度¶

    衡量一个块内语义一致性的指标。

    // 计算块内嵌入的方差（方差越小，语义越一致）
func IntraChunkCohesion(embeddings [][]float32) float64 {
    if len(embeddings) == 0 {
        return 0
    }

    // 计算平均向量
    centroid := make([]float64, len(embeddings[0]))
    for _, emb := range embeddings {
        for i, v := range emb {
            centroid[i] += float64(v)
        }
    }
    for i := range centroid {
        centroid[i] /= float64(len(embeddings))
    }

    // 计算方差
    var variance float64
    for _, emb := range embeddings {
        dist := cosineDistance(emb, centroid)
        variance += dist * dist
    }

    return variance / float64(len(embeddings))
}

    2. 块间差异度¶

    衡量不同块之间差异性的指标。

    // 计算块间平均距离
func InterChunkDiversity(chunks []Chunk) float64 {
    var totalDist float64
    count := 0

    for i := 0; i < len(chunks); i++ {
        for j := i + 1; j < len(chunks); j++ {
            dist := cosineDistance(chunks[i].Embedding, chunks[j].Embedding)
            totalDist += dist
            count++
        }
    }

    return totalDist / float64(count)
}

    3. 检索召回测试¶

    // 测试分块效果的召回率
func TestChunkingRetrieval(query string, expectedDoc string, chunks []Chunk) float64 {
    // 1. 找到包含 expectedDoc 的块
    relevantChunks := 0
    for _, chunk := range chunks {
        if strings.Contains(chunk.Content, expectedDoc) {
            relevantChunks++
        }
    }

    // 2. 检索相关块
    queryEmb := embed.Query(query)
    retrieved := 0
    for _, chunk := range chunks {
        if cosineSimilarity(queryEmb, chunk.Embedding) > 0.8 {
            if strings.Contains(chunk.Content, expectedDoc) {
                retrieved++
            }
        }
    }

    if relevantChunks == 0 {
        return 0
    }

    return float64(retrieved) / float64(relevantChunks)
}

    分块最佳实践¶

    1. 保留结构信息¶

    // 分块时保留标题层级信息
type Chunk struct {
    Content     string

    2. 动态 ChunkSize¶

    根据内容类型调整分块大小。

    func AdaptiveChunking(text string, docType string) []string {
    var chunkSize int
    var overlap int

    switch docType {
    case "faq":
        chunkSize = 400
        overlap = 50
    case "contract":
        chunkSize = 800
        overlap = 100
    case "report":
        chunkSize = 1200
        overlap = 200
    case "code":
        chunkSize = 600
        overlap = 0
    default:
        chunkSize = 800
        overlap = 100
    }

    return RecursiveChunking(text, chunkSize, overlap)
}

    3. 块质量检查¶

    // 分块后质量检查
func ValidateChunks(chunks []Chunk) []error {
    var errors []error

    for i, chunk := range chunks {
        // 检查块是否为空
        if len(strings.TrimSpace(chunk.Content)) == 0 {
            errors = append(errors, fmt.Errorf("chunk %d is empty", i))
        }

        // 检查块是否太大
        if len(chunk.Content) > 2000 {
            errors = append(errors, fmt.Errorf("chunk %d is too large: %d chars", i, len(chunk.Content)))
        }

        // 检查块是否太小
        if len(chunk.Content) < 20 {
            errors = append(errors, fmt.Errorf("chunk %d is too small: %d chars", i, len(chunk.Content)))
        }
    }

    return errors
}

    总结¶

    分块策略 适用场景 优缺点
    固定大小 通用场景 简单但可能截断句子
    句子级 FAQ、对话 语义完整但大小不均
    段落级 文章、报告 可读性强但大小差异大
    递归分块 复杂文档 智能但实现复杂
    语义分块 高精度场景 质量高但成本高

    选择分块策略需要根据具体场景权衡，没有万能的最佳方案。
