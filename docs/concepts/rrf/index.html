    倒数排名融合 (RRF)¶

    什么是 RRF？¶

    RRF（Reciprocal Rank Fusion，倒数排名融合）是一种简单而有效的多检索结果融合算法。它不依赖具体的相似度分数，而是基于不同检索结果中的排名进行融合，因此对不同评分体系具有天然的适应性。

    一句话理解¶

    RRF = 把"第一名"的价值给最高，"第二名"次之，以此类推

    RRF 在混合检索中的位置¶

    flowchart LR A[用户查询] --> B[向量检索] A --> C[BM25 检索] A --> D[其他检索] B --> E[排名列表 A] C --> F[排名列表 B] D --> G[排名列表 C] E --> H[RRF 融合] F --> H G --> H H --> I[综合排名] I --> J[Top-K 结果]

    为什么需要 RRF？¶

    多检索结果融合的挑战¶

    不同检索方法使用不同的评分体系：

    检索方法 分数范围 分数含义
    向量检索 0-1（余弦相似度） 语义相似程度
    BM25 0-~1000+ 词项匹配程度
    BM25F 0-~500 带字段权重的 BM25

    问题：直接加权融合会产生偏差，因为分数范围不同。

    RRF 的优势¶

    优势 说明
    无需归一化 直接使用排名，不依赖分数
    跨系统融合 可融合任意评分体系的结果
    对异常值鲁棒 不受极端分数影响
    简单高效 计算复杂度 O(n)

    RRF 算法详解¶

    算法公式¶

    \[RRF\_score(d) = \sum_{i=1}^{k} \frac{1}{r_i(d) + k}\]

    其中： - \(d\) 是文档 - \(r_i(d)\) 是文档 \(d\) 在第 \(i\) 个检索结果列表中的排名（从 1 开始） - \(k\) 是平滑常数（通常设为 60）

    算法原理¶

    flowchart LR A1[Doc A: 排名 1] --> C[RRF 得分] A2[Doc B: 排名 3] --> C A3[Doc C: 排名 5] --> C B1[Doc C: 排名 1] --> D[RRF 得分] B2[Doc A: 排名 4] --> D B3[Doc B: 排名 6] --> D C --> E[总得分] D --> E E --> F[Doc A: 0.032] E --> G[Doc B: 0.031] E --> H[Doc C: 0.032]

    计算示例（k=60）：

    文档 向量检索排名 BM25 排名 RRF 得分
    Doc A 1 4 1/(60+1) + 1/(60+4) = 0.032
    Doc B 3 6 1/(60+3) + 1/(60+6) = 0.031
    Doc C 5 1 1/(60+5) + 1/(60+1) = 0.032

    结论：Doc A 和 Doc C 并列第一，Doc B 第二。

    平滑常数 k 的影响¶

    参数 k 控制融合的平滑度：

    k 值 效果 适用场景
    k=0 完全基于排名，不平滑 极端情况
    k=60 标准平滑（默认值） 通用场景
    k=100 更强调排名差异 需要明显区分度
    k→∞ 趋近于平均排名 需要平滑融合
    flowchart LR A[第1名: 1分] --> B[得分 = 1] C[第2名: 0.5分] --> B D[第1名: 0.016] --> E[得分平滑] F[第2名: 0.016] --> E

    RRF 实现¶

    Go 实现¶

    // RRF 融合多个检索结果
func RRFusion(results []SearchResult, k int) map[string]float64 {
    scores := make(map[string]float64)

    for _, result := range results {
        for rank, docID := range result.Docs {
            // 排名从 1 开始
            position := rank + 1
            scores[docID] += 1.0 / (float64(position) + float64(k))
        }
    }

    return scores
}

// 带权重加权的 RRF
func WeightedRRFusion(results []WeightedSearchResult, k int) map[string]float64 {
    scores := make(map[string]float64)

    for _, result := range results {
        weight := result.Weight // 该检索源的权重
        for rank, docID := range result.Docs {
            position := rank + 1
            scores[docID] += weight * (1.0 / (float64(position) + float64(k)))
        }
    }

    return scores
}

    Python 实现¶

    def rrf_score(doc_id: str, results_list: list, k: int = 60) -> float:
    """计算单个文档的 RRF 得分"""
    score = 0
    for results in results_list:
        if doc_id in results:
            rank = results[doc_id]  # 排名从 1 开始
            score += 1 / (k + rank)
    return score

def rrf_fusion(results_list: list, k: int = 60) -> list:
    """融合多个检索结果"""
    # 获取所有文档
    all_docs = set()
    for results in results_list:
        all_docs.update(results.keys())

    # 计算每个文档的 RRF 得分
    scores = {}
    for doc_id in all_docs:
        scores[doc_id] = rrf_score(doc_id, results_list, k)

    # 按得分排序
    sorted_docs = sorted(scores.items(), key=lambda x: x[1], reverse=True)

    return [doc_id for doc_id, _ in sorted_docs]

    RRF 变体¶

    1. 加权 RRF (Weighted RRF)¶

    为不同检索源分配不同权重：

    \[WeightedRRF\_score(d) = \sum_{i=1}^{k} w_i \cdot \frac{1}{r_i(d) + k}\]
    // WeightedRRF 实现
func WeightedRRFusion(results []WeightedResult, k int) map[string]float64 {
    scores := make(map[string]float64)

    for _, result := range results {
        for rank, docID := range result.Docs {
            position := rank + 1
            // 权重 × 标准 RRF
            scores[docID] += result.Weight * (1.0 / (float64(position) + float64(k)))
        }
    }

    return scores
}

    2. 位置加权 RRF¶

    根据文档在原始结果中的位置给予额外奖励：

    func PositionWeightedRRF(results []SearchResult, k int, boostTopN int) map[string]float64 {
    scores := make(map[string]float64)

    for _, result := range results {
        for rank, docID := range result.Docs {
            position := rank + 1
            baseScore := 1.0 / (float64(position) + float64(k))

            // 对 Top-N 给予额外奖励
            if position <= boostTopN {
                baseScore *= 1.5
            }

            scores[docID] += baseScore
        }
    }

    return scores
}

    3. 置信度加权 RRF¶

    根据检索结果的置信度调整权重：

    type SearchResultWithConfidence struct {
    Docs       []string
    Confidences []float64  // 每个结果的置信度
}

func ConfidenceWeightedRRF(results []SearchResultWithConfidence, k int) map[string]float64 {
    scores := make(map[string]float64)

    for _, result := range results {
        for i, docID := range result.Docs {
            position := i + 1
            confidence := result.Confidences[i]
            scores[docID] += confidence * (1.0 / (float64(position) + float64(k)))
        }
    }

    return scores
}

    RRF 与其他融合方法对比¶

    方法 原理 优点 缺点
    RRF 排名融合 简单、鲁棒、跨系统 不考虑分数差异
    分数加权平均 归一化后加权 考虑分数差异 需要归一化、对异常值敏感
    Borda Count 排名积分 综合考量 计算复杂
    Condorcor 两两对比 理论严谨 计算复杂度高

    2. 处理重复文档¶

    func RRFusionDeduplicate(results []SearchResult, k int) ([]string, map[string][]string) {
    scores := RRFusion(results, k)

    // 按得分排序
    sorted := sortByScore(scores)

    // 去重，记录来源
    final := make([]string, 0)
    sources := make(map[string][]string)

    for _, docID := range sorted {
        if _, exists := sources[docID]; !exists {
            final = append(final, docID)
            sources[docID] = getSources(docID, results)
        }
    }

    return final, sources
}

    3. 与 MMR 结合¶

    // RRF + MMR 多样性重排
func RRFWithMMR(rrfResults []string, query string, embeddings Embedder, lambda float64) []string {
    var selected []string
    var remaining = make([]string, len(rrfResults))
    copy(remaining, rrfResults)

    for len(selected) < topK && len(remaining) > 0 {
        bestScore := -1.0
        bestDoc := ""

        for _, doc := range remaining {
            // RRF 得分（排名）
            rrfRank := float64(indexOf(doc, rrfResults))
            rrfScore := 1.0 / (rrfRank + 60)

            // MMR 多样性
            mmrScore := 0.0
            if len(selected) > 0 {
                docEmb, _ := embeddings.Embed(doc)
                maxSim := 0.0
                for _, sel := range selected {
                    selEmb, _ := embeddings.Embed(sel)
                    sim := cosineSimilarity(docEmb, selEmb)
                    maxSim = max(maxSim, sim)
                }
                mmrScore = maxSim
            }

            // 综合得分
            finalScore := lambda * rrfScore - (1-lambda) * mmrScore

            if finalScore > bestScore {
                bestScore = finalScore
                bestDoc = doc
            }
        }

        selected = append(selected, bestDoc)
        remaining = removeFromSlice(remaining, bestDoc)
    }

    return selected
}

    总结¶

    要点 说明
    核心思想 基于排名而非分数的融合
    关键参数 k=60 是默认平滑常数
    主要优势 简单、鲁棒、跨系统
    变体 加权 RRF、位置加权、置信度加权
    最佳实践 选择合适的 k 值、必要时去重、结合 MMR

    RRF 是混合检索中不可或缺的融合算法，它的简单性和有效性使其成为工业界的首选方案。
