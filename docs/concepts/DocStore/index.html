    DocStore（文档存储）¶

    什么是 DocStore？¶

    DocStore（文档存储）是 RAG 系统中用于存储原始文档及其元数据的组件。与 VectorStore 专注于向量检索不同，DocStore 存储的是完整的文档内容、原始文本和丰富的元数据信息。

    一句话理解¶

    DocStore = 存储"完整文档"的地方

    DocStore 在 RAG 系统中的位置¶

    flowchart LR A[VectorStore] --> B[DocStore] A[向量] B[原始文档] C[用户查询] --> D[向量检索] D --> E[获取文档ID] E --> F[DocStore查询] F --> G[返回完整文档]

    为什么需要 DocStore？¶

    VectorStore 的局限性¶

    问题 说明
    存储碎片化 向量库只存储分块后的片段
    元数据受限 向量库的元数据支持有限
    上下文不完整 检索到的片段缺乏完整上下文

    DocStore 的核心价值¶

    价值 说明
    完整文档存储 保留原始文档的完整内容
    丰富元数据 支持复杂的 JSON 元数据
    上下文恢复 可以根据片段追溯完整上下文
    双向索引 支持从实体到原文的反向检索

    DocStore 核心应用场景¶

    1. 父子文档检索（Parent Document Retrieval）¶

    这是 DocStore 最经典的应用场景：

    flowchart LR A[原始文档] --> B[切分为子块] B --> C[向量化子块] C --> D[VectorStore] B --> E[存储父文档] E --> F[DocStore] G[查询] --> H[向量检索] H --> I[获取子块] I --> J[查询父文档ID] J --> K[DocStore获取父文档]

    痛点与解决： - 痛点：为提高检索精度，文档需要切得很碎（200 tokens） - 解决：DocStore 存储完整父文档，检索到子块后可召回父文档

    在 GoRAG 中的实现：

    // 父子文档检索逻辑
func (r *ParentDocRetriever) Retrieve(ctx context.Context, query string) ([]core.Chunk, error) {
    // 1. 向量检索获取子块
    subChunks, err := r.vectorStore.Search(ctx, query, topK)
    if err != nil {
        return nil, err
    }

    // 2. 获取父文档ID
    parentDocIDs := extractParentIDs(subChunks)

    // 3. 从 DocStore 获取完整父文档
    parentDocs, err := r.docStore.GetDocuments(ctx, parentDocIDs)

    // 4. 组装返回结果
    return assembleChunks(subChunks, parentDocs), nil
}

    2. 元数据关联与动态过滤¶

    DocStore 可以存储复杂元数据，支持高级过滤：

    元数据类型 示例 用途
    权限信息 {"acl": ["admin", "user_group_1"]} 访问控制
    时间信息 {"created_at": "2024-01-01", "expired": false} 时效性过滤
    业务规则 {"category": "product", "status": "active"} 业务逻辑过滤
    关联引用 {"references": ["doc_1", "doc_2"]} 关联查询

    应用场景：

    // 基于权限的过滤
func (r *DocStore) FilterByACL(docIDs []string, userID string) []string {
    var allowed []string
    for _, id := range docIDs {
        doc, _ := r.GetDocument(id)
        if hasAccess(doc.Metadata.ACL, userID) {
            allowed = append(allowed, id)
        }
    }
    return allowed
}

    3. 图谱与原文的双向索引¶

    在 GraphRAG 中，DocStore 扮演着"原文还原"的关键角色：

    flowchart LR A[实体节点] --> B[获取 doc_id] B --> C[DocStore 查询] C --> D[提取原始段落] D --> E[回答"为什么"问题]

    价值： - 图节点是抽象的，需要原文来验证关系 - 回答"为什么 A 和 B 相关"这类解释性问题

    4. 长文本摘要与压缩¶

    在需要处理超长文档时，DocStore 支持按需提取和压缩：

    // 长文档压缩流程
func (c *ContextCompressor) CompressFromDocStore(ctx context.Context, docID string, query string) (string, error) {
    // 1. 从 DocStore 获取完整文档
    fullDoc, err := c.docStore.GetDocument(docID)
    if err != nil {
        return "", err
    }

    // 2. 使用 LLM 生成摘要
    summary := llm.Summarize(fullDoc.Content, query)

    return summary, nil
}

    DocStore vs VectorStore 对比¶

    维度 DocStore VectorStore
    存储内容 原始文档、完整文本 向量、文档片段
    检索方式 ID 精确查询、过滤 向量相似度检索
    元数据 支持复杂 JSON 支持简单 kv
    主要用途 文档还原、上下文补充 语义检索
    性能 快速精确 依赖向量引擎

    DocStore 最佳实践¶

    1. 文档 ID 设计¶

    // 建议的文档 ID 格式
type DocumentID struct {
    Source     string // 数据源
    Type       string // 文档类型
    Timestamp  int64  // 时间戳
    Sequence   int    // 序列号
}

func (id DocumentID) String() string {
    return fmt.Sprintf("%s_%s_%d_%d", id.Source, id.Type, id.Timestamp, id.Sequence)
}

    2. 元数据索引策略¶

    元数据类型 索引方式
    数值型 B-Tree 索引
    字符串型 Hash 索引
    日期型 范围索引
    数组型 倒排索引

    3. 一致性维护¶

    // 确保 DocStore 和 VectorStore 的一致性
func (r *RAGSystem) IndexDocument(doc core.Document) error {
    // 1. 先存储到 DocStore
    if err := r.docStore.Store(doc); err != nil {
        return err
    }

    // 2. 再向量化并存储到 VectorStore
    chunks := r.chunker.Chunk(doc)
    embeddings, _ := r.embedding.Embed(chunks)

    for i, chunk := range chunks {
        chunk.Vector = embeddings[i]
        chunk.ParentDocID = doc.ID
        if err := r.vectorStore.Store(chunk); err != nil {
            return err
        }
    }

    return nil
}

    总结¶

    DocStore 是 RAG 系统中不可或缺的组件：

    1. 核心价值：完整文档存储、丰富元数据、双向索引
    2. 主要应用：父子文档检索、元数据过滤、图谱原文还原、长文本压缩
    3. 最佳实践：合理的 ID 设计、适当的索引策略、保持与 VectorStore 的一致性
