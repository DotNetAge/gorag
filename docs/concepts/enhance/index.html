    检索增强理论 (Retrieval Enhancement)¶

    什么是检索增强？¶

    检索增强（Retrieval Enhancement）是 RAG 系统中用于提升检索质量的各种技术手段。它发生在用户查询和向量检索之间或之后，通过优化查询、扩展检索策略、融合多结果等方式，让系统能够更准确地找到用户真正需要的内容。

    一句话理解¶

    检索增强 = 让"找东西"变得更聪明的一系列技术

    检索增强在 RAG 中的位置¶

    flowchart LR A[用户查询] --> B[查询优化] B --> C[检索增强] C --> D[向量检索] D --> E[结果重排] E --> F[生成答案] G[查询扩展] H[混合检索] I[重排序] J[上下文压缩] C -.-> G C -.-> H E -.-> I D -.-> J

    为什么需要检索增强？¶

    检索面临的根本问题¶

    问题 描述 示例
    语义鸿沟 用户的表达与文档不一致 "退钱" vs "退款流程"
    查询模糊 用户问题表述不清晰 "那个政策"指代不明
    知识断层 缺少相关关键词 专业术语vs通俗表达
    长尾分布 大量低频查询 冷门问题

    检索增强的价值¶

    价值 说明
    弥补语义鸿沟 通过查询扩展、HyDE 等技术桥接用户与文档的表述差异
    提升召回率 混合检索、多查询策略增加相关文档被找到的概率
    提高精度 重排序、上下文压缩让最相关的内容排在前面
    优化上下文 压缩无关信息，让 LLM 更高效地利用检索结果

    检索增强核心策略¶

    1. 查询优化类¶

    1.1 查询扩展 (Query Expansion)¶

    通过添加相关词汇或短语来扩展用户查询：

    flowchart LR A[原始查询<br/>"苹果手机"] --> B[查询扩展] B --> C["苹果手机<br/>OR iPhone<br/>OR Apple手机<br/>OR 苹果公司产品"] C --> D[向量检索]

    实现方式： - 同义词扩展：添加查询词的同义词、近义词 - 相关词扩展：添加领域相关的词汇 - LLM 扩展：用 LLM 生成扩展查询

    1.2 HyDE (Hypothetical Document Embeddings)¶

    让 LLM 先"猜测"可能的答案，再用这个猜测去检索：

    用户问题："如何申请美国签证"

↓ LLM 生成假设性回答

假设回答："申请美国签证需要准备以下材料：护照、签证申请表、照片..."
（这个回答会被向量化）

↓ 用假设回答去检索

找到真实的相关文档

    核心思想：用"答案"去找"答案的来源"

    1.3 查询重写 (Query Rewriting)¶

    直接改写用户查询，使其更适合检索：

    原始查询 重写后
    "那个手机好用吗" "智能手机推荐 评价"
    "昨天那个会议内容" "2024年产品评审会议记录"
    "怎么处理退款" "退款流程 步骤 方法"

    2. 检索策略类¶

    2.1 混合检索 (Hybrid Search)¶

    结合多种检索方法：

    flowchart TB A[用户查询] --> B[向量化] A --> C[关键词] B --> D[向量检索] C --> E[BM25/TF-IDF] D --> F[结果融合] E --> F F --> G[最终结果]
    检索方法 优势 劣势
    向量检索 语义理解 可能遗漏精确匹配
    关键词检索 精确匹配 无法处理同义词
    混合检索 兼具体验和精确 需要融合策略

    2.2 RRF (Reciprocal Rank Fusion)¶

    倒数排序融合：将多个检索结果列表合并：

    # RRF 公式
score(d) = Σ 1 / (k + rank_i(d))

# k 通常取 60
# rank_i(d) 是文档 d 在第 i 个检索结果中的排名

    优点： - 不需要训练，直接使用 - 对不同检索方法都有效 - 可以融合任意数量的结果列表

    2.3 多查询检索 (Multi-Query Retrieval)¶

    用 LLM 生成多个相关查询，同时检索：

    原始查询："什么是机器学习"

↓ LLM 生成多个查询
- "机器学习的定义"
- "机器学习基本概念"
- "machine learning 入门"
- "机器学习能做什么"

↓ 每个查询都进行检索

↓ RRF 融合结果

    3. 结果优化类¶

    3.1 重排序 (Re-ranking)¶

    用更精准的模型对初步检索结果重新排序：

    flowchart LR A[查询] --> B[粗排召回 Top-100] B --> C[Cross-Encoder 重排] C --> D[精排 Top-10] D --> E[LLM 生成]

    Bi-Encoder vs Cross-Encoder：

    方法 特点 速度 精度
    Bi-Encoder 预计算向量 快 中
    Cross-Encoder 实时计算相似度 慢 高

    3.2 上下文压缩 (Context Compression)¶

    压缩检索到的内容，去除无关信息：

    原始文档（2000字）:
"本文介绍机器学习。第一段...（省略100字）...机器学习是人工智能的一个分支...
（省略800字）...常用的机器学习算法包括：1. 线性回归...（具体算法）..."

↓ 压缩后（300字）
"机器学习是人工智能的一个子领域，常用算法包括线性回归、决策树、支持向量机等。
它在分类、回归、聚类等任务中广泛应用。"

    实现方式： - 基于 Prompt 的压缩 - 基于信息密度的选择 - MMR (Maximal Marginal Relevance) 优化

    3.3 父文档检索 (Parent Document Retrieval)¶

    检索时用小块，找到后用大块：

    flowchart TB A[小 chunk 向量库检索] --> B[召回相关小 chunk] B --> C[DocStore 获取父文档] C --> D[返回完整上下文]

    4. 迭代增强类¶

    4.1 多跳检索 (Multi-Hop Retrieval)¶

    需要多次检索才能回答的问题：

    问题："谁创办了阿里巴巴？他的净资产是多少？"

↓ 第一次检索
找到：马云创办了阿里巴巴

↓ 第二次检索（基于第一次结果）
找到：马云净资产约XXX亿美元

    4.2 Agentic RAG¶

    让 Agent 自主决定检索策略：

    flowchart TD A[用户问题] --> B{Agent 判断} B -->|需要检索| C[执行检索] C --> D{结果足够?} D -->|是| E[生成答案] D -->|否| F[改写查询] F --> C B -->|需要工具| G[调用其他工具]

    检索增强技术对比¶

    技术选择矩阵¶

    场景 推荐技术 复杂度 效果提升
    同义词多 混合检索 低 ⭐⭐⭐
    表述差异大 HyDE 中 ⭐⭐⭐⭐
    模糊查询 查询重写 中 ⭐⭐⭐⭐
    精确要求高 重排序 中 ⭐⭐⭐⭐
    上下文长 上下文压缩 高 ⭐⭐⭐⭐
    复杂推理 多跳/Agentic 高 ⭐⭐⭐⭐⭐

    组合策略¶

    flowchart TD A[基础 RAG] --> B[+ 查询优化] B --> C[+ 混合检索] C --> D[+ 重排序] D --> E[+ 上下文压缩] E --> F[高级 RAG] style A fill:#e8f5e9 style F fill:#c8e6c9

    检索增强最佳实践¶

    1. 渐进式增强¶

    不要一开始就用所有技术，建议顺序：

    第1步：基础向量检索
    ↓
第2步：添加混合检索（关键词+向量）
    ↓
第3步：添加查询扩展/HyDE
    ↓
第4步：添加重排序
    ↓
第5步：添加上下文压缩

    2. 评估驱动¶

    每添加一个技术，需要评估：

    指标 说明
    召回率 相关文档是否都被找到
    精确率 返回的结果是否相关
    延迟 额外技术带来的延迟
    成本 LLM 调用的额外成本

    3. 场景适配¶

    场景 重点技术
    客服机器人 查询重写 + 混合检索
    知识库问答 HyDE + 重排序
    复杂推理 多跳检索 + Agentic
    长文档摘要 上下文压缩 + 父文档检索

    常见问题与解决¶

    问题1：检索增强带来额外延迟¶

    解决： - 异步执行（非关键路径） - 缓存扩展查询结果 - 降低重排序的候选数量

    问题2：LLM 扩展查询质量不稳定¶

    解决： - Prompt 优化 - 采样多个扩展查询 - 人工审核扩展规则

    问题3：混合检索结果冲突¶

    解决： - RRF 融合 - 调整各路检索的权重 - 基于历史数据学习融合参数

    总结¶

    检索增强是提升 RAG 系统质量的关键：

    1. 查询优化：让系统更好地理解用户想要什么
    2. 检索策略：用多种方法确保找到相关内容
    3. 结果优化：让最好的结果排在最前面
    4. 迭代增强：处理复杂问题的必要手段

    检索增强不是"越多越好"，而是需要根据具体场景选择合适的组合，并通过评估持续优化。
