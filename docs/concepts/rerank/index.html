    结果重排（Reranking）¶

    什么是 Reranking？¶

    Reranking（重排序）是 RAG 系统中的一种检索后处理技术，位于向量检索（或混合检索）之后、LLM 生成之前。它的作用是对初步检索返回的候选文档进行更精确的排序，将最相关的内容排在最前面。

    一句话理解¶

    Reranking = 对检索结果进行"精细化排序"

    Reranking 在 RAG 中的位置¶

    flowchart LR A[用户查询] --> B[向量检索] B --> C[初步结果 Top-K] C --> D[Reranking] D --> E[精排结果 Top-N] E --> F[LLM 生成] F --> G[最终答案] style D fill:#fff3e0

    为什么需要 Reranking？¶

    向量检索的局限性¶

    问题 说明 示例
    语义近似不等于相关 向量相似度高的文档不一定真正回答问题 "苹果"和"苹果手机"向量接近，但查询是水果时则不相关
    多义性 同一个词有不同含义 "Java"可以是咖啡或编程语言
    长尾查询 冷门问题匹配不佳 专业领域的具体问题

    Reranking 的价值¶

    价值 说明
    更精确的排序 使用更复杂的模型重新打分
    过滤噪音 排除不相关的候选文档
    提升生成质量 让 LLM 看到更相关的内容

    Reranking 核心方法¶

    1. Cross-Encoder Reranking¶

    原理：将查询和文档一起输入模型，计算精确的相关性分数。

    flowchart LR A[查询 + 文档] --> B[联合编码] B --> C[相关性分数]

    特点： - 精度最高 - 速度较慢（需要逐个计算） - 适合小规模候选集的重排

    代表模型： | 模型 | 提供商 | 特点 | | ---------------------- | ------ | -------------- | | bge-reranker-base | BAAI | 开源、高性价比 | | bge-reranker-large | BAAI | 高精度 | | Cohere Rerank | Cohere | 商业效果好 | | OpenAI Rerank | OpenAI | 通用性强 |

    代码示例：

    from FlagEmbedding import FlagReranker

reranker = FlagReranker('BAAI/bge-reranker-base')

# 对候选文档重排序
query = "如何申请退款"
documents = [
    "退款流程：请登录账户 → 订单管理 → 申请退款",
    "苹果是 一种水果，富含维生素",
    "本公司产品退货政策：7天内可申请退货"
]

scores = reranker.compute_score([query, doc] for doc in documents)
# 返回相关性分数列表

    2. LLM-based Reranking¶

    原理：使用大语言模型判断文档与查询的相关性。

    flowchart LR A[查询] --> B[构建 Prompt] B --> C[LLM 逐个判断] C --> D[相关性判定]

    优点： - 理解能力强，可处理复杂语义 - 可同时进行事实核查

    缺点： - 成本较高 - 延迟较大

    3. Keyword-based Reranking¶

    原理：基于关键词匹配程度进行重排。

    flowchart LR A[提取查询关键词] --> B[计算文档关键词覆盖率] B --> C[按覆盖率排序]

    适用场景： - 简单快速的 reranking - 作为其他方法的补充

    Reranking 最佳实践¶

    1. 两阶段检索策略¶

    flowchart LR A[查询] --> B[粗排（向量检索）] B --> C[候选集 100-500] C --> D[精排（Reranking）] D --> E[最终 Top-10] E --> F[LLM 生成]
    阶段 方法 返回数量 速度
    粗排 向量检索 100-500 快
    精排 Cross-Encoder 10-20 较慢

    2. Reranking 模型选择¶

    场景 推荐模型
    通用场景 bge-reranker-base
    高精度需求 bge-reranker-large
    商业产品 Cohere Rerank
    多语言 multilingual-reranker

    3. 过滤而非重排¶

    当候选文档相关性普遍较低时，可以设置阈值直接过滤：

    # 过滤低相关性文档
filtered_docs = [doc for doc, score in zip(docs, scores) if score > 0.5]

    Reranking vs 其他技术¶

    技术 作用阶段 特点
    向量检索 检索阶段 快速、大规模
    Reranking 检索后 精确、小规模
    HyDE 查询优化 改进查询表达
    Context Compression 生成前 压缩上下文

    总结¶

    Reranking 是提升 RAG 系统精度的重要技术：

    1. 核心方法：Cross-Encoder、LLM-based、Keyword-based
    2. 典型流程：粗排 → 精排 → 生成
    3. 选择建议：两阶段检索，Cross-Encoder 作为精排主力
