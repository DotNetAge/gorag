    文档解析 (Document Parsing)¶

    什么是文档解析？¶

    文档解析是 RAG 系统的"入口"，负责将各种格式的原始文档（PDF、Word、HTML、Markdown 等）转换为纯文本，供后续的分块和向量化处理使用。

    一句话理解¶

    文档解析 = 把各种格式的文档"翻译"成纯文本

    文档解析在 RAG 中的位置¶

    flowchart LR A[原始文档] --> B[文档解析] B --> C[纯文本] C --> D[数据清洗] D --> E[分块处理] E --> F[向量化] F --> G[向量数据库] style A fill:#ffcdd2 style B fill:#fff3e0 style C fill:#e8f5e9

    文档解析是 RAG 流水线的第一步，如果解析质量差，后续处理再好也无法弥补。

    为什么需要文档解析？¶

    现实世界的文档格式¶

    格式 特点 解析难度
    PDF 排版复杂，可能有图片、表格 ⭐⭐⭐⭐⭐
    Word (.docx) 结构化较好，但有多级标题 ⭐⭐⭐
    HTML 标签嵌套，需提取正文 ⭐⭐⭐
    Markdown 结构简单，解析相对容易 ⭐⭐
    Txt 纯文本，最简单 ⭐

    解析失败的后果¶

    解析失败 → 内容丢失 → 检索不到 → 回答错误
    • PDF 解析失败可能导致整页内容丢失
    • 表格解析错误可能导致数据错乱
    • 标题层级丢失会影响后续分块效果

    常见文档格式解析¶

    1. PDF 解析¶

    PDF 是企业文档最常见的格式，解析难度最大。

    解析挑战¶

    挑战 说明 示例
    文本抽取 从渲染后的内容提取文本 扫描 PDF 需要 OCR
    版式分析 识别标题、正文、页眉页脚 多栏排版
    表格提取 保持表格结构 合并单元格、跨页表格
    图片处理 提取图片或保留图片位置 图表、签名

    PDF 解析工具对比¶

    工具 语言 优点 缺点
    pdfminer Python 纯 Python，适合文本提取 速度慢
    PyMuPDF (fitz) Python/C 速度快，支持修改 表格处理弱
    pdfplumber Python 表格处理优秀 速度较慢
    paddleocr Python OCR 能力强 资源消耗大
    unstructured Python 多格式统一接口 依赖多

    GoRAG 中的 PDF 解析实现¶

    // PDF 解析器接口
type PDFParser interface {
    Parse(ctx context.Context, reader io.Reader) ([]core.TextBlock, error)
    ExtractMetadata(ctx context.Context, reader io.Reader) (core.Metadata, error)
}

// 实现示例：基于 PyMuPDF 的 PDF 解析
func (p *PDFParser) Parse(ctx context.Context, reader io.Reader) ([]core.TextBlock, error) {
    // 1. 打开 PDF 文档
    doc, err := fitz.Open(reader)
    if err != nil {
        return nil, fmt.Errorf("open pdf: %w", err)
    }
    defer doc.Close()

    var blocks []core.TextBlock

    // 2. 逐页解析
    for i := 0; i < doc.NumPage(); i++ {
        page := doc.Page(i)

        // 3. 提取文本块（保持位置信息）
        text, err := page.GetText("text")
        if err != nil {
            continue
        }

        // 4. 分析文本结构（判断标题、正文）
        blocks = append(blocks, p.analyzeBlocks(text, i)...)

        // 5. 提取图片（可选）
        images, _ := page.GetImageList()
        for _, img := range images {
            blocks = append(blocks, p.processImage(img, i)...)
        }
    }

    return blocks, nil
}

    2. Word 文档解析¶

    Word 文档 (.docx) 结构相对清晰，主要解析挑战在于：

    解析要点¶

    要点 说明 处理方法
    标题层级 识别 h1-h6 对应关系 解析 XML 样式
    列表结构 有序/无序列表 保留缩进和编号
    表格处理 保持行列结构 XML 表格节点解析
    超链接 提取链接文本和 URL 解析 rels 关系

    GoRAG 中的 Word 解析¶

    // Word 解析器实现
func (p *WordParser) Parse(ctx context.Context, reader io.Reader) ([]core.TextBlock, error) {
    // 1. 解压 docx（本质是 zip）
    zipReader, err := zip.NewReader(reader, 0)
    if err != nil {
        return nil, err
    }

    // 2. 读取 document.xml（主内容）
    var docXML io.ReadCloser
    for _, f := range zipReader.File {
        if f.Name == "word/document.xml" {
            docXML, err = f.Open()
            break
        }
    }

    // 3. 解析 XML 获取段落和表格
    body, _ := p.parseDocumentXML(docXML)

    // 4. 按标题层级组织内容
    return p.buildBlockTree(body), nil
}

    3. HTML 解析¶

    HTML 文档的解析重点在于提取正文，去除导航栏、广告等干扰内容。

    解析策略¶

    策略 说明 适用场景
    标签过滤 只保留 p, h1-h6, article 等标签 简单页面
    CSS 选择器 根据 class/id 选择正文区域 复杂页面
    基于 ML 使用模型识别正文区域 通用性要求高 
    // HTML 解析实现
func (p *HTMLParser) Parse(ctx context.Context, reader io.Reader) ([]core.TextBlock, error) {
    // 1. 解析 HTML
    root, err := html.Parse(reader)
    if err != nil {
        return nil, err
    }

    // 2. 使用 CSS 选择器提取正文
    selector := "article, .content, .post-content, main"
    contentNodes := p.querySelectorAll(root, selector)

    // 3. 提取文本并保持结构
    var blocks []core.TextBlock
    for _, node := range contentNodes {
        text := p.extractText(node)
        level := p.getHeadingLevel(node)
        blocks = append(blocks, core.TextBlock{
            Content:     text,
            HeadingLevel: level,
        })
    }

    return blocks, nil
}

    结构化文档解析¶

    表格提取¶

    表格是企业文档中的重要信息载体，解析难度较高。

    flowchart LR A[PDF 中的表格] --> B[表格检测] B --> C[行列分割] C --> D[单元格提取] D --> E[结构重建] E --> F[CSV/JSON 输出]

    处理策略： - 检测表格边框线 - 分析行列对齐方式 - 处理合并单元格 - 输出结构化格式

    // 表格提取示例
func (p *PDFParser) extractTable(page *fitz.Page) ([][]string, error) {
    // 1. 找到页面上的表格区域
    tables := p.tableDetector.Detect(page)

    for _, table := range tables {
        // 2. 提取行列数据
        rows, err := p.tableExtractor.Extract(page, table.Bounds)
        if err != nil {
            continue
        }

        // 3. 处理合并单元格
        normalized := p.normalizeMergedCells(rows)

        return normalized, nil
    }

    return nil, nil
}

    多级标题解析¶

    文档的层级结构对于后续分块至关重要。

    层级 Markdown HTML
    一级 # h1
    二级 ## h2
    三级 ### h3
    ... ... ... 
    // 多级标题树结构
type HeadingNode struct {
    Level    int
    Text     string
    Children []*HeadingNode
    Content  []core.TextBlock
}

func (p *DocParser) buildHeadingTree(blocks []core.TextBlock) *HeadingNode {
    root := &HeadingNode{Level: 0, Text: "ROOT"}
    current := root

    for _, block := range blocks {
        if block.HeadingLevel > 0 {
            node := &HeadingNode{
                Level:    block.HeadingLevel,
                Text:     block.Content,
                Children: []*HeadingNode{},
            }

            // 找到正确的父节点
            for current.Level >= node.Level {
                current = current.Parent
            }
            current.Children = append(current.Children, node)
            node.Parent = current
            current = node
        } else {
            current.Content = append(current.Content, block)
        }
    }

    return root
}

    增量解析与全量解析¶

    全量解析¶

    每次索引重建时，解析所有文档。

    适用场景：
- 首次构建索引
- 数据量不大（< 10万篇）
- 文档格式发生变化

    增量解析¶

    只解析新增或修改的文档。

    // 增量解析策略
func (p *Indexer) IndexIncremental(ctx context.Context, changes []Change) error {
    for _, change := range changes {
        switch change.Type {
        case ChangeAdd, ChangeUpdate:
            // 解析新文档或更新文档
            blocks, err := p.parser.Parse(ctx, change.Reader)
            if err != nil {
                return err
            }
            // 只索引变化的文档
            if err := p.indexBlocks(ctx, blocks, change.DocID); err != nil {
                return err
            }

        case ChangeDelete:
            // 从索引中删除
            if err := p.deleteFromIndex(ctx, change.DocID); err != nil {
                return err
            }
        }
    }
    return nil
}

    增量 vs 全量对比¶

    维度 增量解析 全量解析
    时间 只处理变化部分 处理全部文档
    资源 资源占用少 资源占用大
    一致性 可能存在临时不一致 始终一致
    适用规模 大规模数据 小规模数据

    文档解析最佳实践¶

    1. 解析质量检查¶

    // 解析质量检查
func (p *Parser) ValidateParsing(doc io.Reader) (*ParseReport, error) {
    report := &ParseReport{}

    // 1. 检查页数
    pageCount, _ := countPages(doc)
    report.PageCount = pageCount

    // 2. 检查提取的字符数
    charCount, _ := countCharacters(doc)
    report.CharCount = charCount

    // 3. 检查图片数量
    imageCount, _ := countImages(doc)
    report.ImageCount = imageCount

    // 4. 计算解析置信度
    if pageCount > 0 {
        report.Confidence = float64(charCount) / float64(pageCount*2000) // 假设每页 2000 字符
    }

    return report, nil
}

    2. 错误处理策略¶

    错误类型 处理策略
    解析失败 记录日志，返回空内容，继续处理其他文档
    部分内容丢失 返回已解析内容，标记警告
    格式不支持 降级为纯文本提取，或跳过

    // 解析结果缓存
type ParseCache struct {
    store cache.Store

func (c *ParseCache) Get(docID string) ([]core.TextBlock, bool) {
    key := fmt.Sprintf("parsed:%s", docID)
    data, found := c.store.Get(key)
    if !found {
        return nil, false
    }

    var blocks []core.TextBlock
    json.Unmarshal(data, &blocks)
    return blocks, true
